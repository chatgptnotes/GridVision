Part 1 — Getting Started

First steps with GridVision: dashboard orientation and project setup

Chapter 1 — Dashboard Overview

After logging in, you land on the Project Hub — the central dashboard of GridVision. This page gives you an at-a-glance view of all your SCADA projects, recent activity, and system health.

1.1 Layout

The dashboard is divided into four zones:

ZoneLocationContents
Navigation SidebarLeftLinks to Projects, Devices, Tags, Alarms, Trends, Reports, AI, Settings. Collapsible.
Top BarTopLogo, search, notifications bell, user avatar & dropdown (Profile, Logout).
Project CardsCenterGrid of project cards with name, description, last-modified date, and quick actions (View / Edit / Settings).
Quick StatsRight / BottomSystem-wide numbers: total tags, active alarms, devices online, historian storage used.

1.2 Navigation Sidebar

The sidebar is your primary navigation. Items are grouped logically:

  • Dashboard — return to the project hub
  • Projects — list & manage projects
  • Devices — add & configure communication devices
  • Tags — create & manage data points
  • Alarms — alarm definitions & live alarm list
  • Trends — real-time & historical trend charts
  • Reports — report templates & generation
  • Import/Export — bulk tag operations & data export
  • Sequences — command sequencing
  • Interlocks — safety interlock definitions
  • SBO — Select-Before-Operate configuration
  • AI — AI modules (forecasting, health, maintenance, power quality)
  • Historian — compression & retention settings
  • Redundancy — failover configuration
  • Audit Log — activity history
  • Settings — application & profile settings
💡 Tip Click the hamburger icon (☰) at the top of the sidebar to collapse it to icon-only mode, giving more space to the main content area.

The dashboard displays summary widgets:

1.4 Notifications

The bell icon in the top bar shows unread notifications. Notifications are triggered by:

  • New alarms (severity ≥ High)
  • Device connection failures
  • Redundancy failover events
  • Historian storage warnings (>80% full)
  • User login from new IP address

The Mimic Editor is where you design your SCADA single-line diagrams. It is a full-featured graphical editor with drag-and-drop symbols, bus-bar drawing, text annotations, and tag binding.

3.4 Grid & Snapping

The canvas has a configurable grid (default 5 px). Elements snap to the nearest grid intersection when placed or moved. To disable snapping temporarily, hold Alt while dragging.

3.5 Undo History

The editor maintains up to 50 undo steps per session. Every action (place, move, resize, delete, property change) is tracked. Use Ctrl+Z to undo and Ctrl+Y to redo.

⚠️ Unsaved Changes If you close the browser tab with unsaved changes, a confirmation dialog warns you. Always save before leaving the editor.

Chapter 5 — Drawing Bus Bars

Bus bars are the backbone of any single-line diagram — they represent the electrical conductors connecting switchgear and equipment.

5.1 Entering Line Drawing Mode

1

In the symbol palette, click "Bus Bar" (or press B). Your cursor changes to a crosshair.

2

Click on the canvas to set the start point.

3

Click again to set the end point. The bus bar is drawn between the two points.

4

Press Esc to exit drawing mode, or continue clicking to draw more bus bars.

5.2 Axis Snapping

Bus bars automatically snap to the nearest axis (horizontal or vertical) based on the dominant direction of your two click points. If the horizontal distance is greater, you get a horizontal bar; otherwise vertical. This ensures clean, straight lines.

5.3 Bus Bar Properties

PropertyDefaultDescription
Width6 pxThickness of the bus bar line. Use wider (8–10 px) for main buses, thinner (3–4 px) for feeders.
Color#ef4444 (red)Conventional colors: Red = HV (33kV), Blue = MV (11kV), Green = LV (433V).
Start X / Y(auto)Exact start coordinates. Fine-tune in properties panel.
End X / Y(auto)Exact end coordinates.
💡 Tip — Color Conventions Use consistent colors for voltage levels across all pages: Red = 33kV, Blue = 11kV, Green = 415V. This helps operators instantly identify voltage levels.

Chapter 9 — Live Tag Values on Symbols

One of GridVision's key features is displaying live tag values directly on the mimic diagram — just like a real control room display.

9.1 Show/Hide Values Toggle

In the viewer toolbar, there is a "Show Values" toggle button. When enabled:

  • Transformers display HV and LV voltage readings near their terminals
  • Circuit breakers show current through them
  • Generators show power output and frequency
  • Motors show speed and current
  • Meters display their primary measurement

9.2 Value Formatting

Tag values are formatted automatically based on data type and engineering unit:

Data TypeFormatExample
FLOAT2 decimal places by default33.12 kV
INTEGERNo decimals1247 A
BOOLEANOPEN/CLOSED or ON/OFFCLOSED
STRINGRaw textAUTO

9.3 Transformer HV/LV Display

When a transformer symbol has HV and LV voltage tags bound, the viewer shows two value labels:

  • Top label (near HV terminal): e.g., 33.12 kV
  • Bottom label (near LV terminal): e.g., 11.04 kV

If the value goes outside the normal range (±5% of nominal), the label turns amber. If it exceeds ±10%, it turns red.

Chapter 11 — Navigation & Drill-Down

For complex substations with multiple voltage levels, GridVision supports multi-page navigation with drill-down capability.

11.1 Navigation Zones

A Navigation Zone is an invisible clickable area on the canvas that links to another page.

1

In the editor, select "Navigation Zone" from the palette.

2

Draw a rectangle over the area you want to be clickable (e.g., over a feeder group).

3

In properties, set Target Page to the destination page (e.g., "Feeder 1 Detail").

4

In the viewer, clicking that area navigates to the target page.

11.2 Breadcrumbs

As you drill down through pages, a breadcrumb trail appears at the top:

Main SLD → 11kV Switchgear → Feeder 3 Detail

Click any breadcrumb to jump back to that page level.

11.3 Page Transitions

Pages transition smoothly with a fade animation (200ms). The viewer pre-loads adjacent pages for instant navigation.

11.4 Mini-Map

For large pages, a mini-map appears in the bottom-right corner of the viewer. It shows a thumbnail of the entire page with a highlight rectangle showing the current viewport. Click anywhere on the mini-map to jump to that area. Drag the rectangle to pan.

Chapter 14 — External Tag Address Mapping

When a tag has type EXTERNAL, you must configure how GridVision reads (and optionally writes) the value from the physical device.

14.1 Modbus Register Addresses

00001 – 09999Coils (Discrete Outputs)FC01FC05 (single) / FC15 (multiple)1 bit (Boolean)

Chapter 16 — Calculated Tags

Calculated tags derive their values from other tags using JavaScript expressions.

16.1 Formula Field

In the tag creation form, when you select type CALCULATED, a Formula field appears. Enter a JavaScript expression:

// Reference other tags by name in curly braces:

// Power factor from P and S:
{Meter1.Active_Power} / {Meter1.Apparent_Power}

// Three-phase total current:
{Meter1.I_Phase_A} + {Meter1.I_Phase_B} + {Meter1.I_Phase_C}

// Transformer loading percentage:
({TX1.Load_MVA} / 10) * 100    // 10 MVA rated

// Temperature conversion (°F to °C):
({Sensor1.Temp_F} - 32) * 5/9

// Conditional (ternary):
{CB1.Status} ? 1 : 0

// Math functions:
Math.sqrt({Meter1.V_L1} ** 2 + {Meter1.V_L2} ** 2)
Math.max({TX1.Temp}, {TX2.Temp})

16.2 Supported Functions

All standard JavaScript Math functions are available: Math.abs, Math.sqrt, Math.pow, Math.min, Math.max, Math.round, Math.floor, Math.ceil, Math.sin, Math.cos, Math.log, etc.

⚠️ Circular References Do not create circular references (Tag A depends on Tag B which depends on Tag A). The system detects this and marks the tag as ERROR.

19.4 Deadband Explained

Without deadband, an alarm might rapidly toggle on/off when the value hovers near the setpoint (called "chattering"). The deadband prevents this:

// Example: HI alarm, setpoint=85, deadband=2
Temperature rises to 85°C → Alarm ACTIVATES
Temperature drops to 84°C → Alarm still ACTIVE (within deadband)
Temperature drops to 83°C → Alarm CLEARS (crossed 85-2=83)

Chapter 20 — Alarm Banner

20.1 Banner Layout

  • Severity 1–2: Blue/green background — informational

20.3 Expand/Collapse

20.4 Mute

The mute button silences the audible alarm horn. It does NOT acknowledge alarms — just stops the sound. The visual indicators remain. Mute auto-resets after 30 minutes.

🚨 Safety Warning Never ignore persistent alarms. Even if muted, visually monitor the alarm banner. Severity 4–5 alarms may indicate equipment damage or safety hazards.

22.2 Suppression

Suppression is like shelving but applied at the alarm definition level. A suppressed alarm will not activate until unsuppressed. Use suppression when a tag is known to be faulty (e.g., broken sensor).

22.3 Unshelving

Shelved alarms automatically unshelve when the duration expires. To unshelve early, go to the shelved alarms list and click "Unshelve".

⚠️ Shelving Accountability All shelve/unshelve actions are logged with user, time, reason, and duration. Excessive shelving may indicate a poorly tuned alarm system. Review shelved alarms regularly.

The statistics panel shows:
