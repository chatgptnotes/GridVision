1. Platform Overview

GridVision SCADA is a modern, web-based Supervisory Control and Data Acquisition platform designed for monitoring and controlling electrical substations, power distribution networks, and industrial facilities.

🖥️ Mimic Editor

Drag-and-drop SCADA page builder with 90+ electrical symbols, bus bars, and navigation zones.

📡 Protocol Support

Modbus RTU/TCP, OPC UA, DNP3, IEC 61850 — connect to PLCs, RTUs, and meters.

📊 Real-Time Monitoring

Live tag values on mimic diagrams, trend charts, alarm banners, and faceplates.

🤖 AI Analytics

Load forecasting, anomaly detection, predictive maintenance, and power quality analysis.

🛡️ Safety Systems

Interlocking logic, Select-Before-Operate (SBO), command sequencing, authority levels.

💾 Historian

Deadband and swinging door compression, configurable retention, efficient storage.

3. Installation & Setup

Option A: Docker (Recommended — 5 minutes)

1

Install Docker

Download Docker Desktop from docker.com (Windows/Mac) or install via apt (Linux):

# Ubuntu/Debian
$ curl -fsSL https://get.docker.com | sh
$ sudo usermod -aG docker $USER
2

Clone the Repository

$ git clone https://github.com/chatgptnotes/GridVision.git
$ cd GridVision
3

Configure Environment

$ cp .env.example .env
$ nano .env   # Edit with your settings

Key settings to change:

JWT_SECRET=your-unique-secret-key-here
DATABASE_URL=postgresql://gridvision:gridvision_pass@postgres:5432/gridvision_scada
REDIS_URL=redis://redis:6379
PORT=3001
4

Start Everything

$ docker compose up -d

This starts PostgreSQL, Redis, the GridVision server, and the web frontend.

5

Open in Browser

→ http://localhost:5173
✅ That's it! GridVision is now running. Default login: admin@gridvision.local / admin123

Option B: Manual Installation (From Source)

# 1. Install prerequisites
$ sudo apt install -y nodejs npm postgresql redis-server

# 2. Install pnpm
$ npm install -g pnpm

# 3. Clone and install dependencies
$ git clone https://github.com/chatgptnotes/GridVision.git
$ cd GridVision
$ pnpm install

# 4. Setup database
$ createdb gridvision_scada
$ cp .env.example .env
$ nano .env  # Set DATABASE_URL
$ npx prisma migrate deploy
$ npx prisma db seed

# 5. Build and start
$ pnpm build
$ pnpm start

11. Setting Up Alarms

Go to Alarm Config in the sidebar to define alarm thresholds:

  1. Click "+ New Alarm"
  2. Select a tag (e.g., TX1.Winding_Temp)
  3. Set condition: HI (high), HIHI (critical high), LO (low), LOLO (critical low)
  4. Set setpoint (e.g., 85°C for HI, 95°C for HIHI)
  5. Set severity: Low, Medium, High, Critical, Emergency
  6. Set deadband (e.g., 2°C — alarm clears at 83°C, not 84.9°C)
  7. Enable/disable auto-acknowledge, require comment
  8. Save — alarms are now live and will appear in the bottom alarm banner

12. Creating Trend Charts

  1. Go to Trends in the sidebar
  2. Click "+ Add Pen" to add tags to the chart
  3. Select tag, choose color, assign Y-axis (left or right)
  4. Add multiple pens to compare values (e.g., HV voltage + LV voltage + temperature)
  5. Use time range buttons: 1min, 5min, 1h, 24h, 7d, Custom
  6. Pan: drag to scroll time. Zoom: scroll wheel
  7. Save as named trend config for quick access

13. Generating Reports

  1. Go to Reports in the sidebar
  2. Create a template: Shift Report, Daily Generation, Event Log, or Custom
  3. Select tags, time range, aggregation (Average/Min/Max/Sum)
  4. Group by: Hour, Shift (8h), or Day
  5. Click "Generate" → preview the data
  6. Download as CSV or PDF

14. Configuring Interlocks & Safety

Interlocks prevent dangerous operations:

  1. Go to Interlocks in the sidebar
  2. Create interlock: Target tag (e.g., CB1.Status), Target action (SET to true/CLOSE)
  3. Add conditions: Isolator1.Status = OPEN AND Voltage < 1kV
  4. All conditions must be TRUE for the action to be ALLOWED
  5. If conditions fail → action is BLOCKED with explanation

Select-Before-Operate (SBO):

  1. Go to SBO in the sidebar
  2. Enable SBO for critical tags (circuit breakers, isolators)
  3. Set timeout (default 30 seconds)
  4. In the mimic viewer: first click = SELECT (yellow highlight), second click = OPERATE

17. Redundancy & Failover

For critical installations, configure hot standby:

  1. Install GridVision on two servers
  2. Go to Redundancy in the sidebar
  3. Server 1: Set role = PRIMARY, partner = Server 2 IP
  4. Server 2: Set role = STANDBY, partner = Server 1 IP
  5. Enable auto-failover
  6. If primary goes down, standby takes over automatically within 15 seconds

21. Scripting & Action Components

GridVision includes 5 scripting/action component types you can drag onto the canvas:

ComponentPurpose
Action ButtonClickable button that runs a script (e.g., open/close a breaker)
Script RunnerExecutes a script on page load or on a trigger
Formula DisplayShows a calculated value (e.g., total generation = Gen1 + Gen2 + Gen3)
Sequence TriggerRuns a multi-step command sequence
Conditional DisplayShows/hides content based on tag conditions

Simplified Script Syntax

GridVision uses a simple, human-readable script syntax — no semicolons, no quotes needed:

// Basic assignment
CB1_Status = 1
Gen1_Setpoint = 50.5

// Conditional logic
IF Voltage > 240 THEN
  AlarmHigh = 1
ELSE
  AlarmHigh = 0
END

// Math formulas
TotalLoad = Gen1_MW + Gen2_MW + Gen3_MW
Efficiency = (TotalLoad / Capacity) * 100

// Electrical formulas
Power = Voltage * Current * 1.732 * PowerFactor

AI Script Generator

  1. Select an action component on the canvas
  2. In the Properties panel, click the purple "✨ AI Generate" button
  3. Describe what you want in plain English: "Close CB1 if voltage is above 11kV and isolator is open"
  4. AI generates the script using GPT-4o-mini (online) or local rules (offline)
  5. Review and click "Use Script"

Script Reference (F1)

Press F1 in the editor to open the Script Reference popup — 36 entries across 7 categories:

  • Basic Operations, Conditional Logic, Math & Formulas
  • Common Sequences, Display & Conditional, Electrical Formulas, Timing & Triggers
  • Search entries, click "Insert" to paste into your script

23. AI Custom Component Creator

Create your own SCADA components using AI — describe in words or upload a hand-drawn sketch.

Creating a Custom Component

  1. In the Mimic Editor palette, look for the "✨ Custom" category at the top
  2. Click "+ Create Component"
  3. Choose your method:
    • Describe: Type a description like "3-phase motor with cooling fan and temperature sensor" → click Generate
    • Upload Sketch: Take a photo of a hand-drawn component sketch → upload → AI converts to clean SVG
  4. Preview the generated SVG — edit the code if needed
  5. Set the component name, category, dimensions
  6. Configure tag binding points (e.g., status, speed, temperature) — AI suggests these automatically
  7. Click Save

Using Custom Components

  • Saved components appear permanently in the "✨ Custom" palette section
  • Drag and drop onto canvas like any built-in symbol
  • Bind tags to the pre-defined binding points
  • Edit or delete custom components from the palette (hover for ✏️ and 🗑️ icons)
  • Components are per-project — each project has its own custom library

24. Custom Report Designer

Create fully customized reports with a 5-step wizard. Navigate to Report Designer in the sidebar.

Step 1: Data Sources

  • Browse all available tags in a searchable list
  • Click tags to add them as report columns
  • Set aggregation per column: Average, Min, Max, Sum, Last Value, Count
  • Add calculated columns with formulas: TotalGen = Gen1_MW + Gen2_MW + Gen3_MW
  • Reorder columns by dragging

Step 2: Time & Grouping

  • Time grouping: 1 min | 5 min | 15 min | 1 hour | Shift (8h) | Day
  • Group by: None, Substation, Department, Equipment Type
  • Date range: Custom dates or quick presets (Last 1h, 24h, 7 days, 30 days)

Step 3: Filters & Conditions

  • Filters: "Show only rows where Voltage > 220" — supports >, <, ==, !=, >=, <=
  • Highlight rules: "Color the cell RED when Load < 50 MW" — choose from red, orange, yellow, green, blue
  • Multiple filters and rules can be combined (AND logic)

Step 4: Output & Schedule

  • Output type: Table only | Chart only | Table + Chart
  • Chart type: Line or Bar (pure SVG, no external libraries)
  • Export formats: PDF (HTML), Excel, CSV
  • Schedule: Manual | Daily 6 AM | Daily 6 PM | Every Shift | Weekly | Monthly | Custom cron
  • Share reports with team or keep private

Step 5: Preview & Save

  • Live preview with real/simulated data
  • Highlight rules applied visually on cells
  • Summary row with aggregated totals
  • Save as a reusable template
  • "Generate Now" for immediate download

25. Client-Side Tag Simulator

Test your mimic pages without a real SCADA connection using the built-in tag simulator:

  1. Open any page in the Mimic Viewer
  2. The simulator automatically starts, fetching all project tags
  3. A floating values panel appears showing all active tag values in real-time
  4. Tags with simulation patterns (sine, ramp, random) animate with realistic values
  5. Values update in real-time on the canvas — see breakers change color, gauges move, trends update

26. External Device Manager

Connect to real industrial equipment via the Device Manager:

  1. Go to Device Manager in the sidebar
  2. Click "Add Device"
  3. Configure:
    • Name: e.g., "Substation RTU-01"
    • Protocol: Modbus TCP/RTU, OPC UA, DNP3, IEC 61850
    • Connection: IP address, port, unit ID
    • Polling: Scan rate in milliseconds
  4. Map tags to device registers:
    • Address type: Holding Register, Input Register, Coil, Discrete Input
    • Register address (e.g., 40001)
    • Byte order: Big Endian / Little Endian / Big Endian Swap / Little Endian Swap
    • Scaling: Factor and offset (raw × factor + offset = engineering value)

27. Desktop Application (Electron)

GridVision is available as a standalone desktop application with offline capability:

Installation

  • Windows: Download GridVision-Setup.exe from the Downloads page → double-click to install
  • Linux: Download .AppImage or .deb → run directly or install via package manager
  • macOS: Download .dmg → drag to Applications folder

Offline Mode

  • The Electron app includes an embedded server — no internet required after installation
  • Local PostgreSQL and Redis are bundled
  • All features work offline: editor, viewer, tags, alarms, historian
  • Data syncs to the cloud server when connectivity is restored
